# 系统管理联调与验收清单

## 1. 联调范围

本轮联调覆盖前端系统管理模块的 3 个页面及其依赖接口：

1. 用户管理页 `/system/users`
2. 角色权限页 `/system/roles`
3. 审计日志页 `/system/audit`

联调过程中同时验证了这些页面依赖的后端接口：

1. 认证接口：`/api/v1/auth/login`、`/api/v1/auth/me`
2. 部门接口：`/api/v1/system/departments`
3. 权限接口：`/api/v1/system/permissions`
4. 角色接口：`/api/v1/system/roles`
5. 用户接口：`/api/v1/system/users`
6. 密码接口：`/api/v1/system/users/{id}/reset-password`、`/api/v1/system/me/change-password`
7. 审计接口：`/api/v1/system/audit-logs`

## 2. 已修复问题清单

### 2.1 后端系统路由源码混合编码

- 现象：后端启动时报 `SyntaxError` / `unicode error`，服务无法启动。
- 根因：`backend/app/api/v1/routers/system.py` 文件存在混合编码内容。
- 处理：重建为干净的 UTF-8 源码文件。
- 结果：后端已可正常启动，系统管理路由可正常加载。

### 2.2 写接口提交时机过晚，导致“创建后立即查询”偶发 404

- 现象：新建部门成功后，立即调用详情接口会返回 404；随后更新同一条数据又能成功。
- 根因：事务提交依赖 `get_db()` 的 `yield` 清理阶段，响应已返回但事务可能尚未完成提交。
- 处理：将系统管理写接口的提交时机前移到路由函数返回前。
- 结果：新建后立即读取详情已稳定通过。

### 2.3 用户、角色、权限删除链路存在遗留关联问题

- 现象：用户逻辑删除后，角色删除仍提示“仍有用户绑定”。权限删除未校验角色绑定关系。
- 根因：
  1. `SysUserRole` 关联在用户逻辑删除时未清理。
  2. 角色删除前的绑定检查未排除已删除用户。
  3. 权限删除前未校验仍被有效角色使用的情况。
- 处理：
  1. 用户删除时清理角色绑定。
  2. 角色删除时仅统计未删除用户，并清理遗留角色/权限绑定。
  3. 权限删除时增加有效角色绑定校验，并清理遗留角色权限绑定。
- 结果：删除链路已按业务预期工作。

### 2.4 审计日志 `detail` 字段接口契约不一致

- 现象：后端返回 JSON 字符串，前端类型定义为对象，审计页展示会出现字符串化内容。
- 根因：`SysAuditLogResponse.detail` 定义为 `str | None`。
- 处理：在 Pydantic 响应层将 `detail` 解析为对象。
- 结果：审计日志接口现已返回对象结构，符合前端类型约定。
